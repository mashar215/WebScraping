Gitの基本的なコマンド

git clone https-url or ssh-url 

これで他人が作成したソースコードを自分のパソコンに引っ張ってくることができる。

git init 

これは自分の作成したローカルのファイルに対して使用する。
新しくGitリポジトリを作成するために使用する。
実行するとカレントディレクトリに.gitディレクトリが作成されりぽっじとりの設定や履歴情報が格納される。

git add 
.ならカレントディレクトリ以下を、
ファイル名ならそのファイルをステージングエリアに追加する。
弾丸を装填するに近い。

git status 

リポジトリの現在の状況を確認するために使用する。

以下の状況が確認できる。

・ステージングエリアに追加されたファイル
・変更されているが追加されていないファイル
・未追跡のファイル

git commit -m "wawawa"

ステージングエリアに追加された変更をリポジトリに保存するコマンド
メッセージをつけることが推奨される。

git log 

コミット履歴を確認することができる。

以下の情報が確認できる。
・コミットのハッシュ値
・誰がコミットしたか
・日付
・コミットのメッセージ

--oneline,--graphをオプションにすることで、表示形式を変更できる。

git diff

作業ディレクトリとステージングエリア、コミット間の差分を表示するために使用される。
オプションをつけて確認するのが妥当だと思う。

git branch

ブランチの操作を行う。

・新しくブランチを作成する
    → git branch 作成したいブランチの名前
・ブランチを削除する
    → git branch -d 削除したいブランチの名前

git checkout 

ブランチやコミットへの切り替えを行うために使用される。

git checkout ブランチの名前
    → これで特定のブランチに切り替えることができる。

git checkout -b 作成したいブランチの名前
    → これで新しくブランチを作成して、即座にそのブランチに切り替えることができる。

git merge 統合したいブランチの名前

まず取り込みを行いたいブランチに移動する。
そこで
git merge 取り込みたいブランチの名前
を実行すると、
その取り込みたいブランチの変更点が取り込みを行いたいブランチに反映される。

git pull 

リモートリポジトリの変更を取得し、現在のブランチに統合するために使用される。
コマンド自体は

git fetch + git merge

に相当する。


押さえておきたい用語

リポジトリ

２種類ある

・リモートリポジトリ
    → 開発者が自分のパソコン上で変更を行う場所   
・ローカルリポジトリ
    → 複数人が共同で作業をするための中央管理場所的な？

コミット

ユニークなハッシュ値が与えられる。変更を記録する単位。

以下の情報が含まれている。

・変更内容
・コミットメッセージ
・コミットした人の情報

ブランチ

開発の分岐点。
ブランチを作成すると元のブランチからコピーが作成され、その上でブランチ上で変更を加えると
それぞれのブランチが異なるコミットを持つことになる。

ブランチでの作業が終了したらmergeコマンドで変更をメインに統合する。

タグ

特定のコミットに名前を付ける機能。
マイルストーン的な役割を持つ。

マージ

ブランチの変更を統合する操作。

コンフリクト

複数のブランチで同じファイルの同じ箇所が変更された場合に発生する衝突のこと。
開発者が自ら解決する必要がある。

余談

originとは？

よく使われるorigin、その実体はリモートリポジトリ全体を指している。
リモートリポジトリのデフォルトで与えられるニックネームみたいなもの。

だからリモートリポジトリ内の特定のブランチにアクセスする場合、

origin...×
origin リモートブランチ名...〇

が正しい。

だから特定のリモートブランチにプッシュをかける場合は

git push origin プッシュ先のリモートブランチ名

で行ける。

リモートブランチの右に書かれているdefaultの正体とは？
デフォルトブランチという。
メインとなるブランチのことで、リポジトリをクローンしたときに選択されるブランチのこと。

またmergeの対象もこれになるので注意。

ほかのブランチをデフォルトに選択することも可能。


要するにメインブランチだ。





使えるコマンド集

originはリモートリポジトリ全体を指すニックネームです。
適宜読み替えてください。

git branch -r
    → リモートリポジトリ内にあるリモートブランチの一覧を取得することができる。

git remote rename <old> <new>
    → デフォルトではorigin
    → その名前を変更することができる。

git push origin --delete remote-branch-Name
    → リモートブランチを削除することができる。

git pull origin remote-branch-name 
    → リモートブランチの最新情報をローカルブランチに反映する。

git push origin local-branch-name
    → ローカルのブランチをリモートブランチに登録する。
